Overview

Fireworks.ai API Provider Service is an OpenAI and Anthropic compatible API gateway that proxies requests to Fireworks.ai with intelligent key rotation, real-time monitoring, and automatic failover.

Your App ──▶ API Provider Service ──▶ Fireworks.ai (OpenAI SDK) (rotation + monitor) (Inference API)
FeatureDescription
OpenAI CompatibleDrop-in replacement for /v1/chat/completions and /v1/completions
Anthropic CompatibleDrop-in replacement for /v1/messages
Key RotationRound-robin across multiple API keys, ordered by priority
Auto FailoverKeys marked unhealthy after 3 consecutive failures; retried after 5 minutes
MonitoringReal-time request logs, token usage, latency, cost estimation
Backup & RestoreAuto-backup with SHA-256 checksum integrity verification
Key ValidationTest any key with a live request to Fireworks via the Check button
Dashboard AuthJWT-based authentication (HS256, 24h expiry)

Quick Start

1. Install & Run

git clone <repo-url> && cd api-provider-service pip install -r requirements.txt cp .env.example .env python main.py

The service starts on http://localhost:10736. Dashboard at /login with default credentials admin / macro.

2. Add API Keys

Log into the dashboard, go to API Keys, and add your Fireworks.ai keys (starting with fw_). Use the Check button to verify each key works.

3. Use with OpenAI SDK (Python)

from openai import OpenAI client = OpenAI( base_url="http://localhost:10736/v1", api_key="not-needed" # Keys managed by service ) response = client.chat.completions.create( model="accounts/fireworks/models/kimi-k2p6", messages=[{"role": "user", "content": "Hello!"}] ) print(response.choices[0].message.content)

4. Use with Anthropic SDK (Python)

import anthropic client = anthropic.Anthropic( base_url="http://localhost:10736", api_key="not-needed" ) response = client.messages.create( model="accounts/fireworks/models/kimi-k2p6", messages=[{"role": "user", "content": "Hello!"}], max_tokens=256 ) print(response.content[0].text)

5. Use with cURL

curl http://localhost:10736/v1/chat/completions \ -H "Content-Type: application/json" \ -d '{ "model": "accounts/fireworks/models/kimi-k2p6", "messages": [{"role": "user", "content": "Hello!"}] }'
The api_key field can be any value when calling the inference endpoints. The service manages the real Fireworks.ai keys internally via rotation.

Authentication

The inference API endpoints (/v1/*) require no authentication. The dashboard API endpoints (/api/*) require a JWT token in the Authorization header.

Obtaining a Token

POST/api/auth/loginLogin & get JWT

Request Body:

FieldTypeDescription
usernamestringREQUIRED Dashboard username
passwordstringREQUIRED Dashboard password

Response (200):

{ "token": "eyJhbGciOiJIUzI1NiIs...", "username": "admin" }

Using the Token

Include the token in all subsequent dashboard API requests:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

Token Details

PropertyValue
AlgorithmHS256
Expiry24 hours
SecretSet via SECRET_KEY env var

Error Handling

All errors return JSON with an error field. Common error codes:

StatusMeaningCause
400Bad RequestMissing or invalid request body
401UnauthorizedMissing, expired, or invalid JWT token
404Not FoundResource (key, model) does not exist
429Rate LimitedToo many requests from your IP (if rate limiting enabled)
500Server ErrorInternal error processing the request
503No Keys AvailableAll API keys are unhealthy or none configured
{"error": "No API keys available"}
If you get a 503 error, add API keys via the dashboard or check key health. Keys are automatically retried after 5 minutes of being marked unhealthy.

Dashboard Auth API

POST/api/auth/loginAuthenticate

Login and receive a JWT token. No auth required.

Request: { "username": "admin", "password": "macro" }

Response (200): { "token": "eyJ...", "username": "admin" }

Error (401): { "error": "Invalid credentials" }

GET/api/auth/verifyVerify token

Verify if the current JWT token is still valid.

Response (200): { "valid": true, "username": "admin" }

Error (401): { "error": "Token expired" }

POST/api/dashboard/keysCreate key
FieldTypeDescription
namestringREQUIRED Friendly name
api_keystringREQUIRED Full Fireworks API key
is_activebooleanOPTIONAL Default: true
priorityintegerOPTIONAL Higher = used first. Default: 0
PUT/api/dashboard/keys/:idUpdate key

Update any field of an existing key. All fields are optional. The rotator is reloaded automatically.

DELETE/api/dashboard/keys/:idDelete key

Permanently deletes a key. The rotator is reloaded automatically.

POST/api/dashboard/keys/:id/checkValidate a key

Sends a live test request to Fireworks (model: kimi-k2p6, max_tokens=1) to verify the key works.

{ "valid": true, "key_name": "Production Key", "status_code": 200, "latency_ms": 450, "message": "Key is valid and working" }

Returns aggregate health status from the rotator.

Backup & Restore API

POST/api/dashboard/backupCreate backup
FieldTypeDescription
include_logsbooleanOPTIONAL Include request logs (default: false). Warning: can produce large files.

Creates a JSON backup with SHA-256 checksum. Stored in data/backups/.

{ "message": "Backup created successfully", "filename": "backup_20250115_103000.json" }
GET/api/dashboard/backup/listList backups
[ { "filename": "backup_20250115_103000.json", "size": 524288, "created_at": "2025-01-15T10:30:00" } ]
POST/api/dashboard/backup/restoreRestore from backup

Request: { "filename": "backup_20250115_103000.json" }

Restoring overwrites ALL current data. The key rotator is reloaded after restore.

Deployment

Production with Gunicorn

gunicorn backend.app:app \ --bind 0.0.0.0:10736 \ --workers 4 \ --timeout 120 \ --access-logfile - \ --error-logfile -

Production Checklist

  • Change DASHBOARD_PASSWORD from default macro
  • Set a strong SECRET_KEY for JWT signing
  • Enable HTTPS via reverse proxy (nginx/caddy)
  • Restrict security.allowed_origins to your domains
  • Enable rate_limiting.enabled if exposed publicly
  • Set up external backup storage for data/backups/
  • Monitor health via /health or /healthz

Railway / Render

The project includes Procfile and railway.json ready for deployment. Set environment variables in the platform dashboard.

Reverse Proxy Example (nginx)

server { listen 443 ssl; server_name api.example.com; location / { proxy_pass http://127.0.0.1:10736; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_read_timeout 120s; } }
